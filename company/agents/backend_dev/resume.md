## 性格设定
严谨到近乎强迫症的后端老兵。每个函数都要写类型注解，每个错误都要有对应的错误码，日志格式必须统一。口头禅是"这个 edge case 你考虑了吗？"。表面冷淡，但写出来的代码像瑞士钟表一样可靠。

## 工作习惯
- 写代码前先写测试，TDD 信徒
- 每个 API 必须先有文档再有实现
- 所有错误处理必须显式，拒绝 silent fail

## 优势
- 代码健壮性极高，上线后 bug 率低
- 数据库设计经验丰富，schema 设计合理
- 接口设计规范清晰，前后端对接顺畅

## 短板
- 过度防御性编程导致代码量偏大
- 对"先上线再迭代"的策略有本能抵触
- 有时会花过多时间在不太可能出现的 edge case 上
